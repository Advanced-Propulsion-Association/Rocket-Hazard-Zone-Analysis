# PDF Output Design — Hazard Zone Calculator

**Date:** 2026-04-16  
**Status:** Approved  

---

## Summary

Replace the current minimal `window.print()` with a formal two-page PDF report suitable for FAA AST submission. A dedicated `PrintView` component renders a light-themed layout; `@media print` hides the main app and shows only `PrintView`. The Plotly trajectory chart is re-themed to light colors before printing and reverted after. The map is replaced with a static OSM tile snapshot with the hazard circle drawn on canvas.

---

## Decisions Made

| Question | Decision |
|---|---|
| PDF structure | Formal report (B) — title block, inputs summary, results |
| Trajectory chart | Re-theme Plotly to light colors before print, revert after |
| Map | Static OSM tile snapshot — canvas tile stitch + hazard circle |
| Implementation approach | Dedicated `PrintView` component (Option 2) |
| Methodology placement | Top of Page 1, immediately after title block |

---

## Page Structure

### Page 1
1. **Title block** — "HAZARD ZONE ANALYSIS REPORT / Amateur Rocketry — FAA AST Submittal", date, tier label, `Generated by: Hazard Zone Calculator v1.0`, launch site coords, site elevation, max wind, max launch angle
2. **Warnings** — amber banner(s), if any; shown immediately after title block so the reviewer sees them before any numbers
3. **Methodology** — compact paragraph: 3-DOF point-mass (TAOS-equivalent), ballistic descent, CD source, angle/wind sweep, 1976 USSA, hazard radius formula
4. **Input Parameters** — 2-column table, tier-dependent (see below)
5. **Results** — 2×2 card grid: Hazard Zone Radius (blue accent), Maximum Apogee, Worst-case Launch Angle, Motor/Impulse
6. **¼-Altitude Rule Check** — green/amber banner matching screen layout
7. **Stability panel** — Tier 2/3 only; green/amber/red border matching stability category
8. **Page footer** — "3-DOF ballistic model · NAR/Tripoli safety envelope · 1976 US Standard Atmosphere" | "Page 1 of 2"

### Page 2
1. **Launch Site — Hazard Zone Map** — static OSM tile image (canvas-stitched) with red dashed hazard circle and blue launch pin; OSM attribution; fallback coordinates block if fetch fails
2. **Ballistic Trajectory Sweep** — Plotly chart re-themed to light colors (white bg, slate axes, blue worst-case, red dashed hazard line)
3. **Tier 1 Altitude Table** — Tier 1 only; full altitude range table (same data as screen)
4. **Barrowman CD Breakdown** — Tier 3 only, when no OR CSV loaded
5. **OR Apogee Comparison** — shown when `.ork` file was loaded
6. **Page footer** — "Hazard Zone Calculator v1.0" | "Page 2 of 2"

## Input Parameters Table (tier-dependent)

**Tier 1:**
| Field | Value |
| Apogee altitude | X ft |
| Site elevation | X ft MSL |

**Tier 2:**
| Field | Value |
| Rocket diameter | X in (X mm) |
| Body length | X in (X mm) |
| Fineness ratio | X |
| Gross mass | X lb (X g) |
| Motor | designation (Class X) |
| Total impulse | X N·s |
| CD (source) | X × multiplier = effective |
| Stability margin | X cal (category) |

**Tier 3:** Same as Tier 2, plus nose cone type, fin geometry, nozzle exit diameter, CD source (Barrowman / OR CSV / fineness).

## New Files

### `src/components/PrintView.tsx`
- Props: `result: HazardZoneResult`, `launchCoords: {lat,lon} | null`, `windBearing: number | null`, `inputSummary: PrintInputSummary | null`
- Root div must have `id="print-root"` to match the `@media print` CSS rule
- Renders complete two-page layout in plain inline CSS (no Tailwind dark utilities); Plotly chart always uses light colors (white bg, slate axes)
- Conditionally renders tier-specific sections based on `inputSummary.tier` and result fields
- Does **not** contain any screen-interactive elements (no buttons, no state, no toggle)

### `src/utils/mapSnapshot.ts`
Exports `buildMapSnapshot(lat: number, lon: number, hazardRadius_m: number): Promise<string | null>`
- Calculates zoom level so hazard circle fills ~50% of the 580×160 canvas
- Fetches a 3×3 grid of OSM tiles (`tile.openstreetmap.org/{z}/{x}/{y}.png`)
- Stitches tiles on an offscreen `<canvas>`; crops to 580×160 centred on launch coords
- Draws red dashed hazard circle (scaled to pixels at chosen zoom)
- Draws blue filled launch-site dot with ring
- Draws OSM attribution text bottom-left
- Returns `canvas.toDataURL('image/png')`
- On any fetch error: returns `null` (PrintView falls back to coordinates text block)

## Print Button Flow

The existing "Print / Save as PDF" button in `Results.tsx` is replaced with a new handler. No Plotly re-theming needed — `PrintView` renders its own always-light chart; the screen chart is hidden by `@media print`.

```
1. If launchCoords present: await buildMapSnapshot(lat, lon, hazardRadius_m)
   → pass resulting data URL (or null) to PrintView via a callback/ref
2. window.print()
```

If map snapshot returns null, PrintView shows a coordinates fallback block instead of the map image.
